# Strategy Knowledge Base Template

## Purpose

This document serves as both a template and seed library for the AI-driven strategy iteration loop. Each strategy pattern is documented in a standardized format that Claude can read, reference, and build upon during the propose-evaluate-learn cycle.

Store individual entries as `.json` or `.md` files in the `knowledge_base/` directory.

---

## Knowledge Base Maintenance Rules

1. **Never delete failed strategies** — they contain negative knowledge that prevents repeating mistakes.
2. **Tag all entries with source** — distinguish between seeded patterns and iteration-generated learnings.
3. **Update historical_performance** if you re-run a strategy on new data.
4. **Link related_strategies** — helps Claude find connections between approaches.
5. **Review quarterly** — mark strategies as "deprecated" if market regime has changed fundamentally.
