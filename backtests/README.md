# /backtests/ — Gate 1 Verdict Archive

This directory stores Gate 1 backtest verdicts generated by the quant pipeline.

## Directory Structure

```
backtests/
├── README.md                           ← This file
├── <StrategyName>_YYYY-MM-DD.txt       ← Human-readable Gate 1 verdict
├── <StrategyName>_YYYY-MM-DD.json      ← Machine-readable Gate 1 verdict
├── test/                               ← Unit test outputs (NOT real strategy verdicts)
│   └── <TestStrategy>_YYYY-MM-DD.*
└── test-artifacts/                     ← Archived legacy test outputs (historical)
    └── TestMomentum_2026-03-15.*       ← Generated by gate1_reporter.py unit tests
    └── FailStrat_2026-03-15.*
    └── CondStrat_2026-03-15.*
```

## Convention

### Production backtests (root directory)
- All files in the root `/backtests/` directory are **real strategy Gate 1 verdicts**
- Generated by `run_gate1_<strategy>.py` runners
- Named `<StrategyName>_YYYY-MM-DD.{txt,json}`

### Unit test outputs (`test/` subdirectory)
- All unit test output from `gate1_reporter.py` and other test suites **must** write to `/backtests/test/`
- Never write test files to the root `/backtests/` directory
- The `generate_and_save_verdict()` function accepts `output_dir=_BACKTESTS_TEST_DIR` for this purpose

### Legacy test artifacts (`test-artifacts/` subdirectory)
- `TestMomentum_2026-03-15.*`, `FailStrat_2026-03-15.*`, `CondStrat_2026-03-15.*` were generated
  by the unit tests in `gate1_reporter.py` (`_run_tests()` function) before the test/production
  directory separation was established. They were moved here on 2026-03-16.
- These files are preserved for audit trail purposes but are NOT real strategy verdicts.

## Gate 1 Criteria Reference

See `criteria.md` in the repo root for the full Gate 1 acceptance criteria (CEO-locked v1.0).

Pass thresholds:
- IS Sharpe > 1.0
- OOS Sharpe > 0.7
- IS Max Drawdown < 20%
- Win Rate > 50%
- Trade Count > 50 (IS period)
- Parameter Sensitivity < 30% Sharpe change
- Walk-forward: pass ≥ 3 of 4 windows
- DSR z-score > 0
